PolyGEN combines tools for automatic PTG design and automatic prime editing guide RNA design to enable an easy use of polycistronic elements both in the context of prime editing experiments but also for any other setup profiting from polycistron-produced RNAs.

PTG Design Manual

The PTG function offers design automation of polycistronic tRNA-based genes containing custom RNAs for assembly in BsaI-driven Golden Gate experiments. The backbone of the PTG design is based on iBioCAD by Scott Weisberg in HamediRad et al. (2019), for which the code is openly available through Github.

To use the PTG design function of PolyGEN, the user must provide a list of the desired custom RNAs to be included in the finished polycistron of the form:

id0;type0;sequence0
id1;type1;sequence1
...

where 'id' is some specific identifier of the part, 'type' is the type of RNAs, which the part belongs to ('gRNA', 'pegRNA' or 'sRNA') and 'sequence' is the entire sequence of the part in 5' -> 3' direction. Note that 'type' can at this point only be one of the three listed options. Other inputs will lead to errors. An exemplary input would be:

g123;gRNA;CGAGGAGCTGTTCA...
peg456;pegRNA;TATGCTATTCGATC...

From this, it will construct the finished polycistron and calculate all oligomers necessary to produce each singular part via PCR with a plasmid containing a gRNA-tRNA template. The produced parts can then be assembled by Golden Gate. Currently, the produced PTGs can include sgRNAs, pegRNAs and other small RNAs. By default, PolyGEN will use the following parameters, which will be adjustable in future updates:

  • primer melting temperature between 52 and 72 Â°C if possible
  • primer annealing length on template between 12 and 30 nt
  • 'tgcc' and 'gttt' as BsaI restriction overlaps with the vector plasmid
  • no additional BsaI restriction sites in the plasmid

To calculate the primer melting temperatures, PolyGEN uses the same method and parameters as Benchling: SantaLucia (1998).

Prime Editing Short Guide RNAs Design Manual

PolyGEN also features a method for automatic design of pegRNAs for desired edits, including specific point mutations, deletions and insertions. Here, the user must supply the sequence which is supposed to be edited with a large enough sequence range around the edit position (at least 50 nucleotides upstream and downstream of any edit) and information about the edit itself. The following is an input template:

pos0;base0;type0
pos1;pos2;type1
pos3;base1;type2

  • For point mutations (mut), the first input specifies the position of the edit, while the second is one the desired new base
  • For deletions (del), the first input is the starting and the second input the ending position of bases that should be deleted
  • For insertions (ins), the first input is the position of insertion and the second the sequence of bases to be inserted

Note, that currently the positions are specified as the indices of an array, meaning that the first position of a sequence is 0. Note also that similarly to the type input for the PTGs, here the type of mutation can again only be one of "mut", "del" or "ins". Other inputs will lead to errors. Each type of edit can span multiple bases and is not restricted to the point mutation, deletion or insertion of one base per pegRNA. A possible list of inputs could look like this:

28,30,31;A,T,G;mut
104,108;106,111;del
66;CCA;ins

The above input list would produce

  • One pegRNA which changes positions 28 to A, 30 to T and 31 to G
  • One pegRNA which deletes the bases from 104 to 105 (104 and 105) and from 108 to 110 (108, 109 and 110)
  • One pegRNA which inserts the sequence CCA at position 66

Currently, it is not possible to include edits of different types in one pegRNA
